# Operations & Architecture

This directory contains the operational blueprints, technical specifications, and reports for the MyBizAI project. It is designed to facilitate parallel development by multiple teams.

## Directory Structure

*   **`blueprints/`**: High-level architectural designs and product requirements (PRDs) for major features.
*   **`specs/`**: Detailed technical specifications for specific components or APIs.
*   **`reports/`**: Progress reports, post-mortems, and decision logs.

## Current Blueprints

### 1. [Existing Entrepreneur Onboarding](./blueprints/ideation_onboarding_existing.md)
*   **Goal**: A specialized flow for users who already have a business.
*   **Focus**: Optimization, scaling, and problem-solving (vs. ideation).
*   **Status**: Ready for Implementation.

### 2. [Voice Interface](./blueprints/voice_interface.md)
*   **Goal**: Add Speech-to-Text and Text-to-Speech capabilities to the chat.
*   **Status**: Ready for Implementation (Phase 1 MVP).

### 3. [Business Plan Editor (Phase 2)](./blueprints/business_plan_editor.md)
*   **Goal**: A rich-text, AI-assisted editor for drafting business plans.
*   **Status**: Ready for Implementation.

## Usage
Teams should refer to the relevant blueprint before starting work on a feature. Updates to the architecture should be reflected here.
